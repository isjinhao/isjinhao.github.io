浮点数

发表于 2020-01-06 | 更新于 2020-08-10 | 分类于 基础知识
本文字数： 3.2k | 阅读时长 ≈ 3 分钟

在说这个问题之前，我们先看几个基础概念。第一个是n位二进制能最大表示的无符号整数？这个大家应该都知道是$2^0 + 2^1 + … + 2^{n-1}$，即$2^n – 1$。第二个是二进制的科学计数法。大家都知道十进制中有一个科学计数法，即如825.25会被表示成$8.2525 10^2$。显然在二进制的世界中也有一个科学计数法。如8.25在二进制被表示为1000.01，写成科学计数法就可以表示成$1.00001 2^3$。120.5在二进制中为1111000.1，写成科学计数法就是$1.1110001 2^6$。所以大家是不是发现在二进制的科学计数法中浮点数可以被表示成$1.xxx 2^t$的形式呢，这样的话我们也就可以顺道推导出计算机如果要表示一个浮点数会将其分成三个部分：符号位（0为正、1位负，记为S）、尾数部分（1.xxx，记xxx部分为M）和指数部分（t，记为E）。所以二进制的科学计数法就是：$(-1)^S (1.M) 2^E$。

明白这两个概念之后我们就可以来解释Java浮点数的表示范围了。首先Java使用的浮点数运算规则是IEEE 754。那么什么是IEEE 754呢？就是一个美国定义的被广泛采用的浮点数运算标准。在32位的float中分配规则为：1位符号位、8位指数位、23位尾数位。在64位的double中为：1位符号位、11位指数位、52位尾数位。

我们刚才说到的二进制的科学计数法为$(-1)^S (1.M) 2^E$。所以看到这有人肯定会有疑问，E应该也是有正负数的，那为什么不用$(-1)^S$这种形式表示呢？

因为IEEE 754采用了无符号数值减去一个偏正值的形式来存储整数。即若指数部分存储的为00000111，实际上表示的是：00000111 – 偏正值。偏正值为$2^{e-1}-1$（e是指数部分所占的位数）。这种方式被称为指数偏差。这个时候我们就会发现若指数位为8位，则采用了指数偏移形式来存储的指数部分实际表达的数值为[-127, 128]，这竟然违反补码的表示范围了！所以其实指数部分在区间$[1, 2^{e-1}-2]$中才时减去偏正值。而0和$2^{e-1}-1$用于表示一些特别的数值，这个我们在后面再解释。现在更新二进制形式的科学计数法为$ (-1)^S (1.M) 2^{E-偏正值} $。

解释完指数位之后，尾数位就很好理解了，就是m位二进制位表示的无符号整数，区间为$[0, 2^m-1]$。现在我们就可以求出浮点数的范围了。

  • float（32位）：1位符号位（S）、8位指数位（E）、23位尾数位（M）

  • double（64位）：1位符号位（S）、11位指数位（E）、52位尾数位（M）

我们把上面的数值按照$ (-1)^S (1.M) 2^{E-偏正值} $填入就可以得到float和double能表示的数值范围。

其实说到这里我们一直在解释的都是规格形式的浮点数。也就是E的区间为$[1, 2^e-2]$且尾数的整数部分为1的情况。而如果浮点数的指数部分的编码值是0，尾数非零且尾数的整数部分为0，那么这个浮点数将被称为非规约形式的浮点数。但注意：非规格化值域和规格化值域不是互补的，除了这两种还有特殊值！在IEEE 754标准中规定除了规格化情况下之外的其他所有情况偏移后的值为规格情况下偏移后再加1。例如，最小的规约形式的单精度浮点数的指数部分编码值为1，指数的实际值为-126；而非规约的单精度浮点数的指数域编码值为0，对应的指数实际值也是-126而不是-127。实际上非规约形式的浮点数仍然是有效可以使用的，只是它们的绝对值已经小于所有的规约浮点数的绝对值，即所有的非规约浮点数比规约浮点数更接近0。

而至于为什么做的原因是为了解决“突然式下溢出”这个问题（维基百科说是INTEL力荐的）。咱们先来看看不采用IEEE 754标准所产生的一种恶果。假如不采用 IEEE 754，E最小可以取0，以正单精度浮点数举例。最小规格浮点数为$2^{-127}$，次小规格浮点数为$2^{-127} (1 + 2^{-23})$。它俩之间的差值为$2^{-127}2^{-23}$。而最小规格浮点数与0之间的差值为$2^{-127}$。这样就会出现一种情况，两个相近的不相等的规格浮点数做差值时可能得到的数值在0和最小规格浮点数之间，也就是不能被真实表示出来，就会出现突然下降至0的情况。但如果采用了IEEE 754，最小和次小规格浮点数之间的差值为$2^{-126} 2^{-23}$，但在$2^{-126}$至0之间还有能被实际表示出来的浮点数。我们不妨来求一下此时最小的非规格浮点数：$2^{-126} 2^{-23}$。恰好等于相邻规格浮点数之差。所以这种神奇的规定是为了解决这种“突然式下溢出”的情况。

这时候，大家肯定有一个疑问，无论采不采用IEEE 754，能表示的范围都是一定的，为什么会有这种区分呢？实际上就像我们推导出来的float和double范围，超出这个范围的都被认为是非法的情况。在被规定为合法的精度范围内做运算时不会发生错误。
